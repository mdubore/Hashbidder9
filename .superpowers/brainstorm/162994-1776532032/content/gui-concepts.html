Hashbidder GUI Concepts

How should we structure the visual application?

A

Monitor Dashboard (Read-Only + Config Editor)

A web dashboard that monitors your current Braiins, Ocean, and Mempool stats in real-time, with a dedicated "Settings" tab to modify bids.toml visually without running the daemon itself.

B

Active Control Panel (The Daemon Runner)

A GUI application that replaces the CLI. It has a "Start/Stop" button for the daemon loop, real-time logs scrolling on screen, and visual panels to edit the configuration and current active bids on the fly.

C

Native Desktop App (System Tray)

A lightweight system tray application (macOS/Windows/Linux) that runs the Python script in the background. Clicking the tray icon opens a small window showing active bids and a config editor.
